How to write and print a PLAIN TEXT resume?

A plain text resume is perfect for IT workers. It needs some skills to write and print plain text resumes.

Page setting and font selection

The most common paper in North America is Letter which is the most acceptable paper for resumes. The Letter's size is (W)215.9mm X (H)279.4mm. For a resume, the proper side margin is 1 inch(2.54cm). So, the valid width for resume content is 215.9-25.4*2 = 165.1mm.

Since the paper settings are fixed, it's time to choose a proper font family for our resumes. As the resume contains only PLAIN TEXT, monospaced(every character has the same width) font is a must.As for me, "Courier" is my best choice.

Next, we should determine the right font size. Many people suggest that the best font size for a resume is 10pt-14pt. According to the suggestion, I choose Courier 12pt. An important thing worth knowing is that the font size is very different from the font width. When is comes to Courier, font width is about font size multiplying 0.6. As a result, the Courier 12pt font's width is actually 12*0.6=7.2pt=7.2 * (25.4/72)mm=2.54mm. Then we can calculate how many characters can one line contain: 165.1mm / 2.54mm = 65.

As an hacker programmer, you may be used to the the 80x24 screen size. A Letter paper is absolutely different from a screen. So don't write 80(more bigger than 65) characters in one line.

As a computer worker, we often like 64(2^6) more than 65. Finally, I choose 64 as the maximum characters in one line.

SUMMARY:
Paper size: Letter
Pape side margin: left 2.54cm, right 2.54cm
Font name: Courier
Font size: 12pt
Line capacity: 64 characters

Print to a Paper Resume

Right now, I can email my PLAIN TEXT resume everywhere as long as a monospaced font is selected when sending the email. But I also need a physically printed resume which can be brought with me for the future interview. The request for printing is coming.

Directly printing by lpr/lp

Most linux systems now support cups and lp/lpr commands. Since a plain text file contains only text without any format information such as font and page setting, the format information must be given as options of the lp/lpr command. 

lp -o media=Letter \
   -o page-left=72 \
   -o page-top=72 \
   -cpi=10 \
   -lpi=6 \
   resume.txt

NOTE: -cpi=10 #characters per inch(25.4mm/2.54mm=10)

Converting to pdf before printing

If sometimes I cannot use my lovely Linux to print my resume, what should I do? Today even mobile phones can be used to connect to a printer. So, a file able to be printed through all devices is my target.

PDF, as the most protable file format, can not only provide the same view for all screens, but also for printers. Before printing my resume, converting it to a pdf format file is a really good choice.

There are a lot of tools used to generate a pdf file from a plain text file. As my linux OS is CentOS, I only show the built-in tools here. We need 2 apps, enscript (in ghostscript package) and ps2pdf(in enscript package). "enscript" is responsible for converting txt to postscript, while "ps2pdf" is responsible for converting postscript to pdf.

The following cmdline generates resume.ps from resume.txt. 

enscript --no-header \
	--margins=72:72:72:72 \
	--font=Courier12 \
	--media=Letter \
	--output=resume.ps \
	resume.txt

NOTE: --margins=left,right,top,bottom
--font=Courier12: font name Courier, font size 12pt.
The following cmdline converts resume.ps to resume.pdf. 

ps2pdf resume.ps resume.pdf

Now we have the resume.pdf! Printing a pdf file is so easy in all softwares, including Adobe Reader available for almost all devices, and lp/lpr. In fact, using lpr to print a pdf file is exremely simple! 

lp resume.pdf

Because the resume.pdf is ready, besides printing, it can be emailed as an attachment to anyone.
